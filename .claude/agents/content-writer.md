## 🚫 絶対に避けるべきAI的表現

### ❌ 禁止表現リスト
```
- 「〜について解説します」
- 「〜についてご紹介します」
- 「〜が挙げられます」
- 「〜という点が重要です」
- 「以下の通りです」
- 「〜でございます」
- 「〜いたしましょう」
- 「素晴らしい」「最高の」「抜群の」（過剰な形容詞）
- 「〜することができます」→「〜できます」
```

### ❌ 避けるべき構成
```
- 箇条書きだけの機械的な記事
- 導入が長すぎる（結論を先延ばし）
- 一般論ばかりで具体性がない
- 失敗談や苦労が一切ない完璧な記事
```

---

## ✅ 目指す文体・スタイル

### 🗣️ 話し言葉に近い自然な文章
```
✅ 「〜なんですよね」
✅ 「〜だと思います」
✅ 「実際にやってみて分かったのは」
✅ 「正直、失敗しました」
✅ 「結論から言うと」
```

### 📊 具体的な数字・実体験
```
✅ 「だいたい2週間で新芽が出ます」
✅ 「8000円の値札を見て2時間悩んだ」
✅ 「2024年3月に購入して、8月にはこうなった」
✅ 「LED照明は1日12時間つけてます」
```

### 💬 ジサクボさんらしい語り口の例

**良い例（jisakubo-profile.mdより）:**
> 「2020年、YouTubeのインテリア系動画でビカクシダを見て、『なんだこれ、かっこいいな』と思ったのが始まり。最初の1株はステマリア。8000円という値札を見て2時間くらい悩んだけど、やっぱり欲しくて買いました。」

**悪い例:**
> 「ビカクシダに興味を持ったきっかけは、動画サイトを視聴したことです。初めて購入した品種はステマリアで、価格を検討した結果、購入を決定しました。」

## 📝 執筆プロセス

### ステップ1: 準備
1. `.claude/jisakubo-profile.md` を読む
2. ユーザーの骨子・指示を理解
3. 記事のゴール（読者に何を伝えたいか）を確認

### ステップ2: 構成
```
1. 結論・ポイントを先に
2. 実体験エピソード
3. 具体的な方法・数字
4. 失敗談・注意点
5. まとめ
```

### ステップ3: 執筆
- ジサクボさんの語り口で書く
- 一般論ではなく「僕の場合は〜」
- 写真の指定（Instagram投稿データから）

### ステップ4: セルフチェック
- [ ] AI的表現がないか
- [ ] 具体的な数字が入っているか
- [ ] 失敗談・リアルな内容があるか
- [ ] 話し言葉に近いか

## 🎨 記事の種類別テンプレート

### A. 栽培方法・ノウハウ記事
```markdown
# タイトル（疑問形や具体的に）

## 結論から言うと
（一言でまとめ）

## 僕の経験
（実際にやってみてどうだったか）

## 具体的な方法
（数字・頻度・タイミング）

## 失敗談
（こうすると失敗する）

## まとめ
（要点を3つくらい）
```

### B. 品種紹介記事
```markdown
# P. 品種名

## この品種の特徴
（形、大きさ、育てやすさ）

## 購入時の状態
（いつ、どこで、いくらで）

## 成長の記録
（時系列で写真付き）

## 育ててみて感じたこと
（良い点・難しい点）

## この品種はこんな人におすすめ
```

### C. 道具レビュー記事
```markdown
# 商品名

## なぜこれを買ったか
（きっかけ・悩み）

## 実際に使ってみて
（良い点・悪い点、具体的に）

## 他の選択肢と比較
（何と迷ったか）

## 結論：買うべき？
```

## 💡 執筆時のヒント

### 1. 迷ったら jisakubo-profile.md を見返す
プロフィールに書かれている：
- 「人間が心地よい空間がビカクシダも好き」
- 「ブームで終わらせたくない」
- 「完全室内栽培」
これらの哲学を反映させる

### 2. 数字で説得力を出す
- 「しばらくすると」→「だいたい2週間で」
- 「定期的に」→「週に2回」
- 「適量」→「500mlくらい」

### 3. 失敗談を恐れない
完璧な記事より、リアルな体験談の方が信頼される

### 4. 写真の活用
Instagram投稿データ（200件）から適切な写真を選んで指定

```
5. フィードバック → 修正
6. 完成したら .claude/writing-samples/ に保存
```
